# STY0001: End-of-line comments

| Property | Value |
|----------|-------|
| **ID** | STY0001 |
| **Category** | Style |
| **Severity** | Warning |
| **Has Code Fix** | Yes |

## Description

End-of-line comments make code harder to read and maintain. Place comments on their own line above the code they describe.

## Non-compliant Code

```csharp
int timeout = 5000; // Timeout in milliseconds // STY0001
```

## Compliant Code

```csharp
// Timeout in milliseconds
int timeout = 5000;
```

## Rationale

- Comments on their own line are easier to read
- Prevents long lines
- More consistent formatting

## When to Suppress

Suppress for brief inline annotations where a separate line would disrupt readability.

```csharp
#pragma warning disable STY0001
#pragma warning restore STY0001
```

## EditorConfig

```ini
dotnet_diagnostic.STY0001.severity = warning
```
