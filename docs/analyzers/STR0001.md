# STR0001: No continue statements

| Property | Value |
|----------|-------|
| **ID** | STR0001 |
| **Category** | Structure |
| **Severity** | Error |
| **Has Code Fix** | No |

## Description

Avoid using `continue` statements in loops. They make control flow harder to follow and can be restructured into clearer logic using filtering or conditional blocks.

## Non-compliant Code

```csharp
foreach (var item in items)
{
    if (item == null)
        continue; // STR0001

    ProcessItem(item);
}
```

## Compliant Code

```csharp
foreach (var item in items)
{
    if (item != null)
    {
        ProcessItem(item);
    }
}
```

Or use LINQ filtering:

```csharp
foreach (var item in items.Where(i => i != null))
{
    ProcessItem(item);
}
```

## Rationale

- Continue statements create non-obvious control flow
- Makes debugging and reasoning about loops harder
- Restructuring often leads to cleaner, more readable code

## When to Suppress

Suppress only when restructuring would make the code significantly less readable (rare).

```csharp
#pragma warning disable STR0001
#pragma warning restore STR0001
```

## EditorConfig

```ini
dotnet_diagnostic.STR0001.severity = error
```
