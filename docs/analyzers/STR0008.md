# STR0008: One type per file

| Property | Value |
|----------|-------|
| **ID** | STR0008 |
| **Category** | Structure |
| **Severity** | Warning |
| **Has Code Fix** | No |

## Description

Each file should contain only one type (class, struct, interface, etc.). This improves code organization and makes files easier to find.

## Non-compliant Code

```csharp
// In a single file:
public class Customer // STR0008
{
}

public class Order // STR0008
{
}
```

## Compliant Code

`Customer.cs`:
```csharp
public class Customer
{
}
```

`Order.cs`:
```csharp
public class Order
{
}
```

## Exceptions

- Nested classes are allowed within their parent class file

## When to Suppress

Suppress for tightly coupled types that are always used together and are small enough to warrant sharing a file.

```csharp
#pragma warning disable STR0008
#pragma warning restore STR0008
```

## EditorConfig

```ini
dotnet_diagnostic.STR0008.severity = warning
```
