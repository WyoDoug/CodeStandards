# STR0004: If-else chains

| Property | Value |
|----------|-------|
| **ID** | STR0004 |
| **Category** | Structure |
| **Severity** | Warning |
| **Has Code Fix** | Yes |

## Description

Excessive if-else chains should be refactored into switch expressions, pattern matching, or lookup tables.

## Non-compliant Code

```csharp
if (status == Status.Active)
{
    result = "Active";
}
else if (status == Status.Pending)
{
    result = "Pending";
}
else if (status == Status.Completed)
{
    result = "Completed";
}
else
{
    result = "Unknown";
}
```

## Compliant Code

```csharp
var result = status switch
{
    Status.Active => "Active",
    Status.Pending => "Pending",
    Status.Completed => "Completed",
    _ => "Unknown"
};
```

## Rationale

- Switch expressions are more concise and readable
- The compiler can verify exhaustiveness
- Easier to add new cases without introducing bugs

## When to Suppress

Suppress when the conditions involve complex, unrelated boolean expressions that don't map to a switch.

```csharp
#pragma warning disable STR0004
#pragma warning restore STR0004
```

## EditorConfig

```ini
dotnet_diagnostic.STR0004.severity = warning
```
