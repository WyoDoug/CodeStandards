# STY0002: Null-forgiving operator

| Property | Value |
|----------|-------|
| **ID** | STY0002 |
| **Category** | Style |
| **Severity** | Error |
| **Has Code Fix** | No |

## Description

The null-forgiving operator (`!`) suppresses null warnings but can hide null reference bugs. Use proper null checks or assertions instead.

## Non-compliant Code

```csharp
string name = GetName()!; // STY0002
```

## Compliant Code

```csharp
var name = GetName();
if (name != null)
{
    // Use name
}

// Or with pattern matching
if (GetName() is { } name)
{
    // Use name
}
```

## Rationale

- The null-forgiving operator silences the compiler without adding safety
- Null reference exceptions can still occur at runtime
- Proper null handling makes intent explicit

## When to Suppress

Suppress only when you can guarantee the value is non-null through external invariants the compiler cannot verify.

```csharp
#pragma warning disable STY0002
#pragma warning restore STY0002
```

## EditorConfig

```ini
dotnet_diagnostic.STY0002.severity = error
```
