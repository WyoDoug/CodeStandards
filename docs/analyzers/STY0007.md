# STY0007: Region naming convention

| Property | Value |
|----------|-------|
| **ID** | STY0007 |
| **Category** | Style |
| **Severity** | Warning |
| **Has Code Fix** | Yes |

## Description

Regions must follow specific naming patterns and encapsulate complete functional units. Generic region names that group by member type are prohibited.

## Non-compliant Code

```csharp
#region Constructors  // STY0007 - too generic
#region Properties    // STY0007 - too generic
#region Methods       // STY0007 - too generic
#region Fields        // STY0007 - too generic
```

## Compliant Code

```csharp
#region SaveCommand command
// Command field, property, Execute, CanExecute all together
#endregion

#region Name property
// Backing field, property, change handlers all together
#endregion

#region CustomerChanged event
// Event field, event property, raise method all together
#endregion

#region Equality members
// Equals, GetHashCode together
#endregion

#region IDisposable implementation
// Interface implementation together
#endregion
```

## Rationale

- Feature-based regions keep related code together
- Generic regions encourage scattering related members across the file
- Named regions serve as documentation of functional units

## When to Suppress

Suppress for legacy code where refactoring region structure is impractical.

```csharp
#pragma warning disable STY0007
#pragma warning restore STY0007
```

## EditorConfig

```ini
dotnet_diagnostic.STY0007.severity = warning
```
