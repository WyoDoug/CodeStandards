# STR0002: Multiple return statements

| Property | Value |
|----------|-------|
| **ID** | STR0002 |
| **Category** | Structure |
| **Severity** | Error |
| **Has Code Fix** | Yes |

## Description

Methods should have a single exit point (one return statement) at the end. Multiple returns make it harder to debug, add logging, or add cleanup logic.

## Non-compliant Code

```csharp
public int Calculate(int x)
{
    if (x < 0)
        return -1; // STR0002

    if (x == 0)
        return 0; // STR0002

    return x * 2;
}
```

## Compliant Code

```csharp
public int Calculate(int x)
{
    int result = x * 2;

    if (x < 0)
    {
        result = -1;
    }
    else if (x == 0)
    {
        result = 0;
    }

    return result;
}
```

## Rationale

- Single exit point makes debugging predictable — set one breakpoint to see all return values
- Easier to add logging or cleanup code
- Consistent pattern across all methods

## When to Suppress

Suppress for very simple methods where a single return would add unnecessary complexity.

```csharp
#pragma warning disable STR0002
#pragma warning restore STR0002
```

## EditorConfig

```ini
dotnet_diagnostic.STR0002.severity = error
```
