# STY0008: Magic strings

| Property | Value |
|----------|-------|
| **ID** | STY0008 |
| **Category** | Style |
| **Severity** | Warning |
| **Has Code Fix** | Yes |

## Description

Magic string literals should be extracted to named constants or added to string resources if they are user-facing.

## Non-compliant Code

```csharp
if (status == "active") // STY0008
{
    SendEmail("admin@example.com"); // STY0008
}
```

## Compliant Code

```csharp
private const string ActiveStatus = "active";
private const string AdminEmail = "admin@example.com";

if (status == ActiveStatus)
{
    SendEmail(AdminEmail);
}
```

## Rationale

- Named constants document intent
- Single point of change when values need updating
- Prevents typo-related bugs from duplicated strings
- User-facing strings belong in resource files for localization

## When to Suppress

Suppress for well-known string patterns like format specifiers or standard identifiers.

```csharp
#pragma warning disable STY0008
#pragma warning restore STY0008
```

## EditorConfig

```ini
dotnet_diagnostic.STY0008.severity = warning
```
