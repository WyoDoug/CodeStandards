# STY0005: Non-nullable string initialized to null

| Property | Value |
|----------|-------|
| **ID** | STY0005 |
| **Category** | Style |
| **Severity** | Warning |
| **Has Code Fix** | Yes |

## Description

Non-nullable strings must not be initialized to null. Use `string.Empty`, a meaningful default, or make the type nullable.

## Non-compliant Code

```csharp
string result = null; // STY0005
```

## Compliant Code

```csharp
// Option 1: Use string.Empty
string result = string.Empty;

// Option 2: Use meaningful default
string result = "Unknown";

// Option 3: Make it nullable if null has semantic meaning
string? result = null;
```

## Rationale

- Respects the nullable reference type contract
- Eliminates a class of null reference exceptions
- Makes intent explicit

## When to Suppress

Suppress when interfacing with APIs that require null initialization patterns.

```csharp
#pragma warning disable STY0005
#pragma warning restore STY0005
```

## EditorConfig

```ini
dotnet_diagnostic.STY0005.severity = warning
```
