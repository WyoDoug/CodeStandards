# STR0005: Return in nested block

| Property | Value |
|----------|-------|
| **ID** | STR0005 |
| **Category** | Structure |
| **Severity** | Error |
| **Has Code Fix** | Yes |

## Description

Return statements inside nested blocks (loops, conditions) make control flow harder to follow. Use result variables instead.

## Non-compliant Code

```csharp
public int FindIndex(int[] items, int target)
{
    for (int i = 0; i < items.Length; i++)
    {
        if (items[i] == target)
            return i; // STR0005
    }
    return -1;
}
```

## Compliant Code

```csharp
public int FindIndex(int[] items, int target)
{
    int result = -1;
    bool found = false;

    for (int i = 0; i < items.Length && !found; i++)
    {
        if (items[i] == target)
        {
            result = i;
            found = true;
        }
    }

    return result;
}
```

## Rationale

- Single exit point makes debugging predictable
- Easier to add post-loop logic
- Consistent with the single-return rule (STR0002)

## When to Suppress

Suppress for LINQ-style query methods where the pattern is well-understood.

```csharp
#pragma warning disable STR0005
#pragma warning restore STR0005
```

