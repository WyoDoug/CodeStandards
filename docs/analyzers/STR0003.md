# STR0003: Return in void method

| Property | Value |
|----------|-------|
| **ID** | STR0003 |
| **Category** | Structure |
| **Severity** | Warning |
| **Has Code Fix** | Yes |

## Description

Void methods should not use `return` statements to exit early. Use structured control flow instead.

## Non-compliant Code

```csharp
public void ProcessData(Data data)
{
    if (data == null)
        return; // STR0003

    // Process data
}
```

## Compliant Code

```csharp
public void ProcessData(Data data)
{
    if (data != null)
    {
        // Process data
    }
}
```

## Rationale

- Consistent control flow patterns
- Easier to add additional conditions
- More explicit logic structure

## When to Suppress

Suppress when the method body is deeply nested and restructuring would reduce readability.

```csharp
#pragma warning disable STR0003
#pragma warning restore STR0003
```

## EditorConfig

```ini
dotnet_diagnostic.STR0003.severity = warning
```
