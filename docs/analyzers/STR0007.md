# STR0007: Suspicious regions

| Property | Value |
|----------|-------|
| **ID** | STR0007 |
| **Category** | Structure |
| **Severity** | Warning |
| **Has Code Fix** | No |

## Description

Regions with suspicious names may indicate code organization issues. Large regions or regions with names suggesting technical debt should be refactored.

## Non-compliant Code

```csharp
#region Hacks // STR0007
// Code that should be refactored
#endregion

#region TODO // STR0007
// Unfinished code
#endregion
```

## Compliant Code

Refactor the code instead of hiding it in a region. Consider extracting to separate classes or methods.

```csharp
#region CustomerValidation
// Well-organized validation logic
#endregion
```

## Rationale

- Regions with suspicious names often hide code that needs attention
- Encourages proper refactoring instead of masking problems

## When to Suppress

Suppress when the region name is flagged incorrectly and the code within is well-organized.

```csharp
#pragma warning disable STR0007
#pragma warning restore STR0007
```

## EditorConfig

```ini
dotnet_diagnostic.STR0007.severity = warning
```
