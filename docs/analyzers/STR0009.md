# STR0009: Nesting depth exceeded

| Property | Value |
|----------|-------|
| **ID** | STR0009 |
| **Category** | Structure |
| **Severity** | Warning (3+ levels), Error (6+ levels) |
| **Has Code Fix** | No |

## Description

Code nesting should not exceed 3 levels (soft limit, Warning) or 6 levels (hard limit, Error). Deep nesting indicates complex logic that should be refactored.

## Non-compliant Code

```csharp
if (condition1)
{
    if (condition2)
    {
        if (condition3)
        {
            if (condition4) // STR0009 - 4 levels deep
            {
                // Too deep
            }
        }
    }
}
```

## Compliant Code

```csharp
if (ShouldProcess(condition1, condition2, condition3))
{
    ProcessCondition4(condition4);
}
```

Or flatten with guard logic:

```csharp
if (!condition1 || !condition2 || !condition3)
{
    return;
}

if (condition4)
{
    // Logic here
}
```

## Rationale

- At 6 levels (24 character indent), code is too complex to reason about
- Deep nesting indicates logic that should be extracted into separate methods

## When to Suppress

Suppress for generated code or unavoidable deeply nested data structures.

```csharp
#pragma warning disable STR0009
#pragma warning restore STR0009
```

## EditorConfig

```ini
dotnet_diagnostic.STR0009.severity = warning
```
